금융 데이터 벤더 연동 / Koscom / 데이터 스펙 기반 서비스 기획

실제 구현 책임이 있었던 데이터 연동·재전달 경험과, 그 위에서 수행한 요구사항 해석·활용 설계 경험 정리

우리은행 AI데이터사업부 역할과 가장 직접적으로 연결되는 축은 외부 금융 데이터 연동 경험과 데이터 스펙을 읽고 서비스 기획으로 연결한 실무 경험입니다. 아래 내용은 직접 구현 책임이 있었던 영역과 기획 지원 경험을 구분해 정리했습니다.

  • 연결 포인트

1. 직접 맡았던 영역

  • DeepSearch에서 외부 금융·레퍼런스 데이터 벤더 연동과 안정적 재전달 체계를 직접 맡았습니다.
  • Koscom 시세 수신을 포함한 데이터 파이프라인을 다뤘습니다.
  • 단순 수집이 아니라, 실제 서비스가 활용 가능한 형태로 데이터가 안정적으로 전달되도록 운영 기반을 관리했습니다.

2. 우리은행 포지션과 맞닿는 이유

  • AI 서비스 기획은 모델 설명보다 먼저, 어떤 데이터가 어떤 제약으로 들어오고 어떻게 서비스에서 쓸 수 있는지 정리하는 작업이 필요합니다.
  • 외부 데이터 소스와 내부 서비스 사이의 해석 비용을 줄여 본 경험은 데이터 활용 설계와 서비스 기획 지원에 직접 연결됩니다.
  • 금융 데이터는 정확성·시의성·정합성이 중요하므로, 데이터 파이프라인을 운영하며 쌓은 관점이 서비스 기획 단계의 현실성을 높여 줍니다.

3. 데이터 스펙 기반 협업 방식

  • 금융기관·공공금융 조직 대상 제품을 다루며 데이터 스펙을 자주 검토했습니다.
  • 요구사항을 단순 전달받는 수준이 아니라, 어떤 데이터가 필요한지, 어떤 형태로 연결되는지, 서비스에서 어디까지 활용 가능한지 해석하는 역할을 반복적으로 수행했습니다.
  • 이 과정은 엔지니어링·기획·사업 조직이 서로 다른 언어를 쓰는 문제를 줄이고, 실행 우선순위를 맞추는 데 기여했습니다.

4. 표현 원칙

  • 직접 구현으로 말할 수 있는 부분
    외부 금융 데이터 벤더 연동, Koscom 시세 수신, 안정적 재전달 체계
  • 기획/협업으로 말할 수 있는 부분
    데이터 스펙 검토, 요구사항 해석, 활용 설계 지원, 협업 조율
  • 피해야 할 과장
    근거 없는 은행 내부 시스템 직접 구축 주장, 증빙 없는 AI 모델 구현 주장
